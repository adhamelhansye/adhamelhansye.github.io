### The Evolution of Web Applications
this chapter talk about Web Applications history and in the early days
  - These were essentially information repo containing static documents and web browsers were invented as a means of retrieving and displaying documents
  - Most Sites did not authenticate users.
  - if an attacker compromised a web server, he would not normally gain access to any sensitive information, because the information held on the server was already open to public view.
  - An attacker would typically modify the files on the server to deface the web site's contents.
### Common Web Applications Functions
 - Shopping (Amazon)
 - Social Networking (MySpace - Facebook)
 - Banking (Citibank)
 - Web Search (Google)
 - Auctions (eBay)
 - Gambling (Betfair)
 - Web Logs (Blogger) 
 - Web mail (Hotmail - Gmail)
 - Interactive information (Wikipedia)

### Benefits of Web Applications 
 - HTTP,the core communications protocol used to access the World Wide Web, is lightweight and connectionless, This provides resilience in the event of communications errors and avoids the need for the server to hold open a network connection to every user as was the case in many client-side applications, HTTP can be tunneled over other protocols,allowing for secure connections.
 - Web applications deploy their user interface dynamically to the browser
 - The core technologies and languages used to develop web applications are relatively simple.

### Web Applications Security
 - The Site is Secure -> it has been designed to use 128-bit secure socket layer (SSL) technology to prevent unauthorized users from viewing any of your information.
#### The Core Security Problem 
 - Users Can Submit Arbitrary Input 
 - Key Problems Factors 
        - Application must accept and process untrusted data that may be malicious.
 - Immature Security Awareness
 - In-House Development 
 - Deceptive Simplicity
 - Rapidly Evolving Threat Profile 
 - Resource and Time Constraints 
 - Overextended Technologies
 
### The Future of Web Applications Security
The future of web application security will increasingly focus on integrating advanced technologies such as artificial intelligence and machine learning to detect and respond to threats in real time. As cyberattacks become more sophisticated, organizations will prioritize adopting zero-trust security models that assume threats could exist both outside and within the network. Additionally, the rise of regulatory requirements and data privacy concerns will drive the need for continuous security assessments and compliance measures, ensuring robust protection of sensitive user data.
